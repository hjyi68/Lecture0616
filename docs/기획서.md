# HJStock — 유튜브 기반 주식 리서치 AI 에이전트 서비스 기획서

> hans17.com 커리큘럼(LLM 통합, 무벡터DB RAG, ElevenLabs STT/TTS·보이스 클로닝, Vercel/Next.js, 프롬프트 인젝션 방어·캐싱)을 전 기술 스택으로 채택한 프로덕션급 에이전트 서비스 기획.

## 1. 서비스 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | HJStock |
| 한 줄 정의 | 유튜브 경제 채널(슈카월드)의 모든 영상을 텍스트화해 RAG로 축적하고, 사용자가 음성으로 대화하며 투자 인사이트를 얻는 AI 에이전트 |
| 데이터 소스 | https://www.youtube.com/@syukaworld 전체 영상 (신규 영상은 매일 1회 자동 수집) |
| 배포 | Vercel (hjstock.vercel.app), Next.js |
| 개발 도구 | Claude Code |
| UI 톤 | 화이트 베이스, 미니멀·하이엔드 (Apple/Linear/Perplexity급 마감 목표) |
| 에이전트 구성 | 오케스트레이션 에이전트 1개 + 정보수집 에이전트 1개 |

## 2. 핵심 사용자 시나리오

1. 사용자가 화면의 **호출 버튼**(보이스 오브)을 누른다.
2. 대화창이 열리고 마이크 입력이 시작된다 → ElevenLabs STT로 텍스트 변환.
3. 오케스트레이션 에이전트가 OpenAI API를 호출, RAG에서 슈카월드 발언을 검색해 근거 기반으로 답변 생성.
4. 답변 텍스트를 ElevenLabs TTS로 합성해 음성으로 재생(동시에 채팅 버블에 텍스트·출처 영상 링크 표시).
5. 사용자는 텍스트로도 동시에 대화 가능(음성 필수 아님).

## 4. 에이전트 설계

### 4.1 오케스트레이션 에이전트 (대화 담당)
- **트리거**: 호출 버튼 클릭 → 세션 시작 / 또는 텍스트 입력 즉시.
- **LLM**: OpenAI API (모델은 설정에서 교체 가능하게 추상화).
- **역할**
  - 사용자 발화 의도 분류(시황 질문 / 특정 종목·테마 질문 / 잡담 등).
  - RAG 검색 쿼리 재작성(질문 → 검색 친화 쿼리) 후 Top-K 청크 회수.
  - 회수된 청크 + 출처(영상 제목, 업로드일, 타임스탬프, URL)를 컨텍스트로 답변 생성.
  - 근거가 부족하면 "슈카월드 영상에서 해당 내용을 찾지 못했다"고 명시(환각 방지).
  - 프롬프트 인젝션 방어: 영상 자막에서 가져온 텍스트는 항상 "참고자료" 역할로만 격리(시스템 지시로 오인되지 않도록 구분 토큰 처리), 출력 전 인젝션 패턴 필터링.
- **음성 I/O**: STT 입력 → 텍스트 처리 → TTS 출력. 호출 버튼이 눌린 동안 실시간 스트리밍 대화.

### 4.2 정보수집 에이전트 (배치 담당)
- **스케줄**: 1일 1회 (Vercel Cron Job).
- **파이프라인**
  1. 채널의 전체 영상 목록 조회 → 마지막 수집 시각 이후 신규 영상만 필터.
  2. 신규 영상 자막(텍스트) 추출.
  3. 의미 단위로 청킹(타임스탬프 보존) → 임베딩 생성.
  4. 메타데이터(영상ID, 제목, 업로드일, 타임스탬프, 청크 텍스트, 임베딩 벡터)를 Vercel Blob에 저장하고 인덱스(JSON/parquet 등) 갱신.
  5. 실행 로그 및 실패 알림(수집 실패 영상 재시도 큐).
- **최초 적재(백필)**: 서비스 오픈 시 채널 전체 과거 영상에 대해 동일 파이프라인을 1회 일괄 실행.

## 5. RAG 설계 (무벡터DB 방식)

- hans17 커리큘럼 방식 채택: 별도 벡터DB 없이 **임베딩 + 코사인 유사도** 직접 계산.
- 저장: Vercel Blob에 `{videoId, title, publishedAt, chunkIndex, startTime, text, embedding}` 형태의 JSON 라인/배열로 적재.
- 검색 시: 인덱스를 메모리(또는 엣지 캐시)에 로드 → 질의 임베딩과 전체 청크 임베딩 간 코사인 유사도 계산 → Top-K 반환.
- 데이터 규모가 커지면(영상 수 누적) 채널 단위 샤딩 또는 사전 클러스터링으로 검색 비용 최적화(추후 단계).
- 캐싱: 자주 쓰는 질의/임베딩에 Prompt Caching 적용해 비용·지연 절감.

## 6. UI/UX 설계

- **컨셉**: 화이트 배경 + 절제된 그레이스케일 포인트, 여백 중심의 하이엔드 디자인.
- **메인 화면 구성**
  - 중앙 상단: 서비스 로고/타이틀, 한 줄 슬로건.
  - 중앙: 원형 **호출 버튼**(마이크 아이콘, 펄스 애니메이션으로 듣는 중/말하는 중 상태 표시).
  - 호출 시 하단/우측에서 대화 패널 슬라이드인 — 채팅 버블, 출처 카드(영상 썸네일+제목+타임스탬프 링크) 표시.
  - 사이드: 최근 업데이트된 영상 피드, 데이터 최신화 시각 배지("오늘 새 영상 3건 반영").
- **설정 화면**
  - OpenAI API Key 입력란(나중에 등록, 마스킹 처리, 로컬 미저장/서버 암호화 저장 중 선택).
  - ElevenLabs API Key 입력란(동일 정책).
  - 음성 선택: 디폴트 "남자 1 / 여자 1"(노멀 보이스) 라디오 선택 + "내 목소리 클로닝" 업로드 플로우(추후 키 등록 시 노출).
- **반응형**: 데스크톱/모바일 모두 동일 임팩트를 주는 1-컬럼 중심 레이아웃.
- **상태 피드백**: 듣는 중 / 생각 중(RAG 검색+LLM 추론) / 말하는 중을 마이크 버튼 애니메이션과 텍스트 캡션으로 구분.

## 7. 음성 인터페이스 (STT/TTS)

- **공급자**: ElevenLabs.
- **키 관리**: 환경변수/설정 화면에 추후 입력 가능하도록 빈 슬롯으로 시작(키 미입력 시 음성 기능 비활성화 + 텍스트 모드로 폴백).
- **기본 보이스**: 남성1, 여성1(ElevenLabs 기본 제공 보이스 중 선택) — 사용자가 토글로 전환.
- **보이스 클로닝**: 사용자가 본인 음성을 업로드/녹음하면 ElevenLabs Voice Cloning API로 커스텀 보이스 생성, 응답 TTS에 적용.
- **레이턴시 전략**: STT는 스트리밍 인식, TTS는 문장 단위 스트리밍 합성으로 체감 지연 최소화.

## 8. 데이터/보안

- **시크릿 관리**: OpenAI Key, ElevenLabs Key는 사용자가 추후 등록 — 등록 전까지 해당 기능 비활성 상태로 명확히 안내.
- **프롬프트 인젝션 방어**: 자막 원문에 포함될 수 있는 지시문 패턴 무력화(역할 태그 분리, system/user/context 영역 엄격 구분).
- **저작권/출처 표기**: 모든 답변에 원본 영상 링크·타임스탬프를 인용해 "슈카월드 발언 요약/해석"임을 명시, 원문 대량 재생산 금지.

## 10. 단계별 추진 계획

1. **0단계**: 기본 Next.js 골격(현재 hjstock.vercel.app) 위에 화이트 UI 셸 + 호출 버튼 목업.
2. **1단계**: 정보수집 에이전트 — 채널 영상 목록/자막 수집 파이프라인 + Blob 저장 + 백필.
3. **2단계**: RAG 검색 모듈(임베딩/코사인 유사도) + 오케스트레이션 에이전트 텍스트 대화(OpenAI API 연동).
4. **3단계**: ElevenLabs STT/TTS 연동(키 등록 UI 포함) + 호출 버튼 실시간 음성 대화.
5. **4단계**: 보이스 클로닝, 출처 카드 UI, 데일리 업데이트 배지 등 마감 디테일.
6. **5단계**: 프롬프트 인젝션 방어·캐싱 적용, 운영 모니터링/재시도 큐 정비.

## 11. 리스크 및 고려사항

- 유튜브 자막 가용성(자동생성 자막 품질, 미제공 영상 처리 방법) 확인 필요.
- 영상 수 누적 시 무벡터DB 검색 성능 한계 — 추후 샤딩/사전 필터링 전략 필요.
- API 키 미등록 상태의 UX(기능 제한 안내) 설계 필요.
- 투자 자문이 아닌 "정보 요약/출처 인용" 임을 명확히 고지(법적 리스크 관리).
